## О продукте кратко

**BarberBook** — мобильное и веб-приложение для записи клиентов в сеть барбершопов.
Основные пользователи:
- **Клиент** — записывается на услугу к конкретному мастеру на выбранное время.
- **Мастер** — видит своё расписание и подтверждает визиты.
- **Администратор** — управляет мастерами, услугами, расписанием филиала.

В рамках этого проекта детально проработан **клиентский сценарий записи**: бронирование свободного слота, отмена записи, перенос на другое время.

---

## Использованные подходы и нотации

- **Сбор требований:** user stories, acceptance criteria (формат Gherkin: Given / When / Then).
- **Моделирование:** UML 2.x (use case, sequence, activity), ER-диаграмма (Crow's Foot notation).
- **Проектирование API:** REST, OpenAPI 3.0.
- **Обработка ошибок:** HTTP-коды + структурированные ошибки в формате RFC 7807 (Problem Details).
- **Инструменты:** Markdown, Mermaid (диаграммы), YAML (OpenAPI).

## Как читать проект

Рекомендую идти по порядку:
1. Начните с [требований](01-requirements.md) — поймёте контекст.
2. Посмотрите [user stories](02-user-stories.md) — что хотят пользователи.
3. Изучите [модель данных](03-data-model.md) и [диаграммы](04-diagrams.md) — как это устроено внутри.
4. Откройте [API-контракт](05-api-contract.md) и `openapi.yaml` — как этим пользоваться разработчикам.
5. В конце — [ошибки](06-errors.md) и [тестирование](07-testing.md).

Файл `openapi.yaml` можно открыть в [Swagger Editor](https://editor.swagger.io/) — скопируйте содержимое и получите интерактивную документацию.
