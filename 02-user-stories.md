# 2. User Stories и Acceptance Criteria

Истории оформлены в формате **«Как [роль], я хочу [действие], чтобы [цель]»**.
Критерии приёмки — в формате **Gherkin** (Given / When / Then), который разработчики и QA могут напрямую использовать для автотестов.

---

## US-1. Просмотр свободных слотов

> **Как** клиент, **я хочу** видеть свободные слоты у мастеров на выбранный день, **чтобы** выбрать удобное время без звонка в барбершоп.

**Приоритет:** Must have
**Размер:** M

### Acceptance Criteria

**Сценарий 1: Получение списка свободных слотов**
```gherkin
Given я авторизован в приложении
  And я выбрал филиал "BarberBook на Абая"
  And я выбрал услугу "Мужская стрижка" длительностью 30 минут
When я запрашиваю слоты на завтра
Then я вижу список свободных слотов по всем мастерам филиала
  And каждый слот содержит: время начала, ФИО мастера, цену услуги
  And слоты отсортированы по времени начала
```

**Сценарий 2: Фильтр по конкретному мастеру**
```gherkin
Given я выбрал услугу и филиал
When я выбираю конкретного мастера "Алихан А."
Then я вижу только свободные слоты этого мастера
  And слоты, которые уже заняты или вне рабочего дня мастера, не отображаются
```

**Сценарий 3: Нет свободных слотов**
```gherkin
Given выбранная дата полностью занята
When я запрашиваю слоты
Then я вижу сообщение "На эту дату нет свободных слотов"
  And мне предлагается выбрать другую дату
```

## US-2. Создание записи

> **Как** клиент, **я хочу** забронировать свободный слот, **чтобы** гарантированно попасть к мастеру в выбранное время.

**Приоритет:** Must have
**Размер:** L

### Acceptance Criteria

**Сценарий 1: Успешное бронирование**
```gherkin
Given я выбрал свободный слот (мастер Алихан, 24 апреля 14:00, услуга "Стрижка")
When я подтверждаю бронирование
Then система создаёт запись со статусом CONFIRMED
  And я получаю ID записи и подтверждение на экране
  And этот слот больше не отображается как свободный
  And запись появляется в разделе "Мои записи" → "Предстоящие"
```

**Сценарий 2: Слот уже занят (race condition)**
```gherkin
Given я выбрал слот
  And другой клиент забронировал этот же слот на 2 секунды раньше
When я подтверждаю бронирование
Then система возвращает ошибку SLOT_ALREADY_TAKEN (409 Conflict)
  And я вижу сообщение "К сожалению, этот слот только что заняли. Выберите другое время"
  And мне показывается обновлённый список свободных слотов
```

**Сценарий 3: Попытка записаться в прошлое**
```gherkin
Given текущее время 15:00
When я пытаюсь забронировать слот на 14:00 того же дня
Then система возвращает ошибку BOOKING_IN_PAST (422)
  And я вижу сообщение "Нельзя записаться на прошедшее время"
```

**Сценарий 4: Конфликт с собственной записью**
```gherkin
Given у меня уже есть запись на 24 апреля 14:00
When я пытаюсь создать вторую запись на 24 апреля с 14:15 до 14:45
Then система возвращает ошибку BOOKING_OVERLAP (409)
  And я вижу сообщение "У вас уже есть запись в это время"
```

## US-3. Отмена записи

> **Как** клиент, **я хочу** отменить свою запись, **чтобы** освободить время, если мои планы изменились.

**Приоритет:** Must have
**Размер:** M

### Acceptance Criteria

**Сценарий 1: Успешная отмена**
```gherkin
Given у меня есть запись на 24 апреля 14:00 в статусе CONFIRMED
  And текущее время 22 апреля 10:00 (до визита больше 2 часов)
When я нажимаю "Отменить запись"
  And подтверждаю действие в диалоговом окне
Then статус записи меняется на CANCELLED
  And слот снова становится свободным
  And запись перемещается в раздел "Мои записи" → "Отменённые"
```

**Сценарий 2: Отмена после окна отмены**
```gherkin
Given у меня есть запись на 24 апреля 14:00
  And текущее время 24 апреля 13:00 (до визита 1 час)
When я пытаюсь отменить запись
Then система возвращает ошибку CANCELLATION_WINDOW_EXPIRED (422)
  And я вижу сообщение "Отменить запись можно не позднее чем за 2 часа до визита. Свяжитесь с барбершопом"
```

**Сценарий 3: Попытка отменить чужую запись**
```gherkin
Given существует запись booking_id=123, принадлежащая другому клиенту
When я отправляю запрос на отмену booking_id=123
Then система возвращает ошибку FORBIDDEN (403)
  And в логах фиксируется попытка доступа к чужой записи
```

**Сценарий 4: Повторная отмена уже отменённой записи**
```gherkin
Given запись уже в статусе CANCELLED
When я отправляю запрос на отмену этой же записи
Then система возвращает ошибку INVALID_STATE (422)
  And я вижу сообщение "Эта запись уже отменена"
```

## US-4. Перенос записи

> **Как** клиент, **я хочу** перенести свою запись на другое время, **чтобы** не терять её, если текущий слот стал неудобен.

**Приоритет:** Should have
**Размер:** L

### Acceptance Criteria

**Сценарий 1: Успешный перенос**
```gherkin
Given у меня есть запись на 24 апреля 14:00 (мастер Алихан)
  And текущее время 22 апреля (до визита >2 часов)
  And слот 25 апреля 16:00 у мастера Алихана свободен
When я выбираю новый слот 25 апреля 16:00
  And подтверждаю перенос
Then старый слот (24 апреля 14:00) освобождается
  And новый слот (25 апреля 16:00) становится занятым этой же записью
  And ID записи не меняется
  And в истории записи фиксируется событие "Перенесена с 24 апреля 14:00 на 25 апреля 16:00"
```

**Сценарий 2: Перенос к другому мастеру**
```gherkin
Given у меня есть запись к мастеру Алихан на 24 апреля 14:00
When я переношу запись на слот мастера Бауыржана на 25 апреля 11:00
Then запись обновляется: master_id меняется на ID Бауыржана
  And в истории сохраняются оба значения
```

**Сценарий 3: Новый слот уже занят**
```gherkin
Given я пытаюсь перенести запись на слот, который только что заняли
When я подтверждаю перенос
Then система возвращает ошибку SLOT_ALREADY_TAKEN (409)
  And моя старая запись остаётся без изменений
  And мне показывается актуальный список свободных слотов
```

**Сценарий 4: Перенос после окна**
```gherkin
Given до визита осталось меньше 2 часов
When я пытаюсь перенести запись
Then система возвращает ошибку RESCHEDULE_WINDOW_EXPIRED (422)
```

## US-5. Просмотр моих записей

> **Как** клиент, **я хочу** видеть список своих записей, **чтобы** помнить о предстоящих визитах и управлять ими.

**Приоритет:** Must have
**Размер:** S

### Acceptance Criteria

**Сценарий 1: Список предстоящих записей**
```gherkin
Given у меня есть 2 записи со статусом CONFIRMED в будущем
  And 1 запись со статусом COMPLETED в прошлом
When я открываю вкладку "Предстоящие"
Then я вижу только 2 предстоящие записи
  And они отсортированы по дате визита (ближайшая — сверху)
  And для каждой доступны действия "Отменить" и "Перенести"
```

**Сценарий 2: Пустой список**
```gherkin
Given у меня нет записей
When я открываю "Мои записи"
Then я вижу заглушку "У вас пока нет записей" и кнопку "Записаться"
```

## Definition of Done (общее для всех историй)

Фича считается сделанной, когда:
- [ ] Все acceptance criteria выполнены и покрыты автотестами.
- [ ] Код-ревью пройдено, изменения вмержены в `main`.
- [ ] Документация API (Swagger) обновлена.
- [ ] Ошибки и граничные случаи обработаны согласно [06-errors.md](06-errors.md).
- [ ] Логирование действий реализовано.
- [ ] Функциональное и регрессионное тестирование проведены, багов уровня Critical/High нет.
- [ ] Фича задеплоена на staging и проверена PO.
