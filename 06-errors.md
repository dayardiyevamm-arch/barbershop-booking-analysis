# 6. Обработка ошибок

Единый справочник ошибок для всей клиентской части API бронирования.

## 6.1. Классификация

Ошибки делятся на **3 категории**:

1. **Валидация входа** (400) — запрос невалиден синтаксически.
2. **Авторизация и доступ** (401, 403) — проблема с идентичностью / правами.
3. **Бизнес-логика** (409, 422) — запрос корректен, но нарушает бизнес-правило.

## 6.2. Таблица ошибок

| Код | HTTP | Сценарий | Сообщение клиенту | Что делать клиенту |
|---|---|---|---|---|
| `VALIDATION_FAILED` | 400 | Невалидное тело запроса (неверный тип, формат даты, отсутствует обязательное поле) | «Проверьте введённые данные» | Проверить формат полей |
| `UNAUTHORIZED` | 401 | Токен отсутствует, невалиден или истёк | «Войдите снова» | Обновить токен / перелогиниться |
| `FORBIDDEN` | 403 | Клиент пытается управлять чужой записью | «Действие недоступно» | Ничего (возможна атака — фиксируем в логах) |
| `BOOKING_NOT_FOUND` | 404 | Записи с таким ID нет | «Запись не найдена» | Обновить список записей |
| `SLOT_ALREADY_TAKEN` | 409 | Запрошенный слот только что был забронирован | «Этот слот уже занят. Выберите другое время» | Обновить список слотов |
| `BOOKING_OVERLAP` | 409 | У клиента уже есть `CONFIRMED` запись в пересекающееся время | «У вас уже есть запись в это время» | Выбрать другое время или отменить старую |
| `BOOKING_IN_PAST` | 422 | `start_at` в прошлом | «Нельзя записаться на прошедшее время» | Выбрать будущее время |
| `INVALID_STATE` | 422 | Попытка отменить/перенести запись не в статусе `CONFIRMED` | «Запись уже отменена или завершена» | Обновить список записей |
| `CANCELLATION_WINDOW_EXPIRED` | 422 | До визита < 2 часов, отмена запрещена | «Отменить можно не позднее чем за 2 часа. Свяжитесь с салоном» | Позвонить в салон |
| `RESCHEDULE_WINDOW_EXPIRED` | 422 | До визита < 2 часов, перенос запрещён | «Перенести можно не позднее чем за 2 часа. Свяжитесь с салоном» | Позвонить в салон |
| `SERVICE_UNAVAILABLE_FOR_MASTER` | 422 | Выбранный мастер не оказывает эту услугу | «Этот мастер не оказывает выбранную услугу» | Выбрать другого мастера или услугу |
| `SLOT_OUTSIDE_WORKING_HOURS` | 422 | Слот вне рабочего дня филиала | «Филиал закрыт в это время» | Выбрать другое время |
| `INTERNAL_ERROR` | 500 | Неожиданная ошибка сервера | «Что-то пошло не так. Попробуйте позже» | Повторить позже, уведомить поддержку |

## 6.3. Структура ответа с ошибкой

Формат единый для всех ошибок:

```json
{
  "error": {
    "code": "CANCELLATION_WINDOW_EXPIRED",
    "message": "Отменить запись можно не позднее чем за 2 часа до визита",
    "details": {
      "booking_id": "bk_9f3e...",
      "start_at": "2026-04-24T09:00:00Z",
      "cancellation_deadline": "2026-04-24T07:00:00Z",
      "current_time": "2026-04-24T08:15:00Z"
    },
    "request_id": "req_abc123xyz"
  }
}
```

**Зачем `details`:** помогает фронтенду показать клиенту точное сообщение («до визита 45 минут») без повторного запроса.

**Зачем `request_id`:** в логах сервера этот ID связан с полной трассировкой запроса. Клиент может сообщить его в поддержку, и инженер быстро найдёт проблему.

## 6.4. Логирование ошибок

**Правила логирования:**

| Уровень | Что логируется |
|---|---|
| `INFO` | Успешные действия: создание/отмена/перенос записи с user_id, booking_id, timestamp |
| `WARN` | Бизнес-ошибки 4xx (валидации, конфликты) — без PII (телефона, имени) |
| `ERROR` | 5xx ошибки, исключения БД, недоступность внешних сервисов |

**Обязательные поля в логе:**
- `request_id`
- `user_id` (если авторизован)
- `endpoint`
- `http_status`
- `error_code` (если ошибка)
- `duration_ms`

**Пример лог-записи (JSON):**
```json
{
  "timestamp": "2026-04-24T08:15:23.142Z",
  "level": "WARN",
  "service": "booking-service",
  "request_id": "req_abc123xyz",
  "user_id": "c1a2b3...",
  "endpoint": "PATCH /bookings/bk_9f3e.../cancel",
  "http_status": 422,
  "error_code": "CANCELLATION_WINDOW_EXPIRED",
  "duration_ms": 42
}
```

## 6.5. Правила для фронтенда

1. **Никогда не показывать** клиенту технические детали (`request_id`, имена полей БД).
2. Для ошибок `409 SLOT_ALREADY_TAKEN` — автоматически перезагружать список слотов.
3. Для `401` — редирект на экран логина.
4. Для `500` — показать общий экран ошибки с кнопкой «Повторить» и приглашением обратиться в поддержку с `request_id`.
5. Сообщения локализованы (русский / казахский / английский) — фронтенд выбирает текст по `error.code`, а не использует `error.message` напрямую.
